# คู่มือการใช้งานระบบ Nuengdeaw Wellness Monitor

## 1. ชื่อระบบ
ระบบนี้ชื่อว่า `Nuengdeaw Wellness Monitor` เป็นระบบติดตามสัญญาณชีวภาพและประเมินภาวะของผู้ใช้งานแบบเรียลไทม์ ผ่านหน้าเว็บแอปพลิเคชัน โดยรับข้อมูลจากอุปกรณ์สวมใส่ (Wearable) หรือจากโหมดจำลองสำหรับทดสอบระบบ

## 2. ระบบนี้ใช้ทำอะไร
ระบบนี้ใช้สำหรับอ่านสัญญาณชีวภาพของผู้ใช้งาน แล้วประเมินสภาวะร่างกายและความพร้อมของผู้ใช้ในขณะนั้น เช่น ภาวะปกติ ภาวะเครียด ภาวะอ่อนล้า ภาวะสับสน ภาวะตื่นตัวสูง หรือภาวะเสี่ยงที่ต้องเฝ้าระวังเป็นพิเศษ

การใช้งานหลักของระบบมีดังนี้

1. ใช้ติดตามสัญญาณชีวภาพแบบทันที
2. ใช้ประเมินภาวะ Wellness ของผู้ใช้งาน
3. ใช้แสดงผลบนหน้าจอให้ผู้ดูแลหรือผู้ใช้งานเห็นได้ชัดเจน
4. ใช้บันทึกผลการสแกนเป็นประวัติ
5. ใช้พิมพ์รายงานสำหรับงานคลินิก
6. ใช้ส่งต่อผลการประเมินไปยังขั้นตอนดูแลถัดไป

## 3. เครื่องหรืออุปกรณ์ที่ระบบรองรับ
ระบบนี้ไม่ได้ยึดติดกับเครื่องเดียว แต่เป็นแพลตฟอร์มที่รองรับอุปกรณ์หลายประเภท โดยในโค้ดกำหนดชนิดอุปกรณ์ไว้ดังนี้

### 3.1 Polar
ใช้กับอุปกรณ์ประเภท `Polar H10` หรือ `Polar Verity Sense`

เหมาะสำหรับ

1. อ่านค่า Heart Rate (HR)
2. อ่านค่า RR interval เพื่อนำไปคำนวณ HRV แบบ RMSSD
3. ใช้งานผ่าน Web Bluetooth

### 3.2 Empatica
ใช้กับอุปกรณ์ `Empatica E4`

เหมาะสำหรับ

1. อ่านค่า HR
2. อ่านค่า HRV
3. อ่านค่า GSR/EDA
4. อ่านค่าอัตราการหายใจ RR
5. เชื่อมต่อผ่าน E4 Streaming Server และ WebSocket

### 3.3 Muse
ใช้กับ `Muse 2` หรือ `Muse S`

เหมาะสำหรับ

1. อ่านสัญญาณ EEG
2. แยกค่า Theta, Alpha, Beta, Gamma
3. ใช้ดูความสัมพันธ์เชิงสมอง เช่น Theta/Alpha Ratio
4. ใช้งานผ่าน Muse SDK

### 3.4 Generic Device
เป็นโหมดสำหรับเชื่อมต่ออุปกรณ์อื่น ๆ ที่ไม่ใช่ Polar, Empatica หรือ Muse โดยต้องพัฒนา mapping สัญญาณเข้าเอง

### 3.5 Mock Mode
เป็นโหมดจำลองข้อมูล ใช้สำหรับทดสอบหน้าจอและการทำงานของระบบโดยไม่ต้องมีอุปกรณ์จริง

หมายเหตุสำคัญ:
จากการตรวจสอบไฟล์ระบบ ปัจจุบันตัวแปร `DEVICE_TYPE` ถูกตั้งเป็น `mock` จึงหมายความว่าในสถานะปัจจุบันระบบยังแสดงข้อมูลจำลองอยู่ ถ้าจะใช้งานกับเครื่องจริง ต้องเปลี่ยนการตั้งค่าไปเป็นอุปกรณ์จริงก่อน

## 4. ต้องเตรียมอุปกรณ์อะไรบ้าง
การเตรียมอุปกรณ์แบ่งตามรูปแบบการใช้งาน

### 4.1 อุปกรณ์พื้นฐานที่ต้องมี

1. คอมพิวเตอร์หรือโน้ตบุ๊ก 1 เครื่อง
2. เว็บเบราว์เซอร์ที่รองรับระบบ โดยเฉพาะกรณีใช้ Web Bluetooth
3. ไฟล์ระบบ `Nuengdeaw_Wellness.html` และไฟล์ JavaScript ที่เกี่ยวข้อง
4. อินเทอร์เน็ตหรือเครือข่ายภายใน หากต้องส่งข้อมูลไปยังระบบกลาง
5. เครื่องพิมพ์ หากต้องการพิมพ์รายงาน

### 4.2 อุปกรณ์สำหรับใช้งานจริงในโหมด Wearable

1. อุปกรณ์วัดชีพจรหรือ HR/HRV เช่น Polar H10
2. อุปกรณ์วัด GSR/EDA และสัญญาณอื่น เช่น Empatica E4
3. อุปกรณ์ EEG เช่น Muse 2 หรือ Muse S
4. Bluetooth ที่พร้อมใช้งานบนเครื่องคอมพิวเตอร์
5. ซอฟต์แวร์หรือ SDK ของอุปกรณ์นั้น ๆ ตามที่ระบบต้องใช้

### 4.3 อุปกรณ์หรือซอฟต์แวร์เพิ่มเติมเฉพาะบางรุ่น

1. ถ้าใช้ Empatica E4 ต้องเปิด `E4 Streaming Server` ก่อน
2. ถ้าใช้ Muse ต้องโหลด `Muse SDK` หรือ `muse-js` ก่อนใช้งาน
3. ถ้าใช้ Polar ต้องใช้เบราว์เซอร์ที่รองรับ Web Bluetooth

## 5. ระบบรับสัญญาณอะไรบ้าง
ระบบนี้ออกแบบมาเพื่อรับสัญญาณชีวภาพและสัญญาณสมอง ดังนี้

### 5.1 สัญญาณชีวภาพหลัก

1. `HR` หรือ Heart Rate
ความหมายคืออัตราการเต้นหัวใจ หน่วยเป็นครั้งต่อนาที

2. `HRV` หรือ Heart Rate Variability แบบ RMSSD
ใช้ดูสมดุลของระบบประสาทอัตโนมัติ และภาวะความเครียด/ความพร้อม

3. `GSR / EDA`
ใช้สะท้อนระดับการตอบสนองของระบบประสาทซิมพาเทติก และระดับความตื่นตัว

4. `RR`
คืออัตราการหายใจ หน่วยเป็นครั้งต่อนาที

### 5.2 สัญญาณ EEG

1. `Theta`
2. `Alpha`
3. `Beta`
4. `Gamma`
5. `Theta/Alpha Ratio`
6. `IAF`
7. `Microstate`

สัญญาณกลุ่มนี้ใช้ประเมินภาระสมอง ความพร้อม ความอ่อนล้า และภาวะด้านการรับรู้

## 6. ระบบประมวลผลอย่างไร
หลังจากอ่านสัญญาณจากอุปกรณ์ ระบบจะทำงานตามลำดับดังนี้

1. อ่านข้อมูลจากโหมดจำลองหรือโหมด Wearable
2. ตรวจสอบคุณภาพสัญญาณและ artifact
3. แยกค่าชีวภาพและคลื่นสมองที่จำเป็น
4. เปรียบเทียบกับเกณฑ์มาตรฐาน
5. ประเมินภาวะของผู้ใช้งาน
6. ตรวจ Critical Alert ที่ต้องเฝ้าระวัง
7. สร้างผลสรุปบนหน้าจอ
8. บันทึกผลลงในระบบเก็บประวัติ
9. เปิดทางเลือกให้พิมพ์รายงานหรือส่งต่อผล

## 7. โหมดการใช้งานของระบบ

### 7.1 Sim Mode
ใช้สำหรับทดสอบระบบโดยไม่ต้องเชื่อมต่ออุปกรณ์จริง ระบบจะสร้างข้อมูลจำลองขึ้นมาเอง

### 7.2 Wearable Mode
ใช้สำหรับอ่านข้อมูลจากอุปกรณ์จริง ระบบจะเรียกข้อมูลจาก WearableDriver และตรวจสถานะการเชื่อมต่อก่อนใช้งาน

## 8. โปรไฟล์การใช้งานของระบบ
ระบบมี 2 โปรไฟล์หลัก

### 8.1 Personal Profile
ใช้กับผู้ใช้งานทั่วไป โดยสามารถอิงค่า baseline ส่วนบุคคลได้ ถ้ายังไม่มี baseline ระบบจะใช้ค่า static fallback

### 8.2 Clinic Profile
ใช้ในงานคลินิก โดยบังคับใช้ `clinic_standard` และสามารถพิมพ์รายงานหรือส่งต่อผลไปยังทีมดูแลต่อได้

## 8.1 คำอธิบายเรื่อง Baseline ให้เข้าใจง่าย
Baseline คือค่าพื้นฐานของผู้ใช้งานในสภาวะพักนิ่ง ใช้เป็นจุดอ้างอิงว่าค่า HR, HRV, GSR และ RR ของคนนั้นปกติอยู่ที่ระดับใด เมื่อระบบวัดครั้งต่อไป จึงสามารถเปรียบเทียบได้ว่าค่าที่เกิดขึ้นต่างจากสภาวะปกติมากน้อยแค่ไหน

### Personal Baseline
Personal Baseline คือ baseline รายบุคคล ระบบจะเก็บค่าตัวอย่างหลายครั้งในช่วงที่ผู้ใช้อยู่ในสภาวะนิ่ง แล้วคำนวณค่าเฉลี่ยและส่วนเบี่ยงเบนมาตรฐานของ HR, HRV, GSR และ RR จากนั้นนำไปสร้าง adaptive threshold สำหรับประเมินผู้ใช้งานคนนั้นโดยเฉพาะ

หลักการในระบบปัจจุบันคือ

1. เก็บ sample ระหว่างช่วง calibration
2. ต้องมีจำนวน sample ขั้นต่ำก่อน finalize
3. คำนวณค่าเฉลี่ยของ HR, HRV, GSR และ RR
4. ผสมค่าพื้นฐานส่วนบุคคลเข้ากับเกณฑ์มาตรฐานเดิม เพื่อสร้าง threshold ที่เหมาะกับบุคคลนั้น

สรุปง่าย ๆ คือ Personal Baseline ช่วยให้ระบบอ่านผลได้ตรงกับแต่ละคนมากขึ้น

### Clinic Standard
`clinic_standard` คือเกณฑ์มาตรฐานกลางของระบบที่กำหนดไว้ตายตัวสำหรับงานคลินิก ใช้ในกรณีที่ต้องการความสม่ำเสมอในการคัดกรองผู้ป่วยหรือผู้รับบริการหลายราย โดยไม่ใช้ baseline ส่วนบุคคลของแต่ละคนมาเปลี่ยนเกณฑ์

ข้อดีของ clinic_standard คือ

1. ใช้มาตรฐานเดียวกันทุกเคส
2. เหมาะกับงานคัดกรองเบื้องต้น
3. เหมาะกับงานที่ต้อง audit หรือทวนสอบย้อนหลัง
4. ทำให้การเปรียบเทียบระหว่างรายทำได้ง่าย

ดังนั้นถ้าเป็นงานทั่วไปหรือการติดตามตนเอง ให้ใช้ Personal Baseline ได้ แต่ถ้าเป็นงานคลินิก ให้ใช้ `clinic_standard` เป็นหลัก

## 9. ขั้นตอนการเตรียมระบบก่อนใช้งาน

1. เปิดไฟล์ `Nuengdeaw_Wellness.html`
2. เลือกโหมดการทำงานเป็น `SIM MODE` หรือ `WEARABLE MODE`
3. เลือกโปรไฟล์ให้ตรงกับการใช้งาน เช่น Personal หรือ Clinic
4. กำหนด Subject ID หรือ Case ID
5. ถ้าใช้งานกับเครื่องจริง ให้เปิดอุปกรณ์และตรวจการเชื่อมต่อก่อน
6. ถ้าใช้งานแบบคลินิกและต้องการส่งข้อมูลตรง ให้กด `SET TARGET` เพื่อกำหนดปลายทาง
7. กด `SCAN` เพื่อเริ่มอ่านสัญญาณ

## 9.1 ขั้นตอนการเชื่อมต่ออุปกรณ์จริงทีละประเภท

### การเชื่อมต่อ Polar H10 / Polar Verity Sense

1. เปิดอุปกรณ์ Polar และตรวจว่าแบตเตอรี่พร้อมใช้งาน
2. เปิด Bluetooth บนคอมพิวเตอร์
3. เปิดระบบผ่านเบราว์เซอร์ที่รองรับ Web Bluetooth
4. เปลี่ยนการตั้งค่าอุปกรณ์ในระบบจาก `mock` ไปเป็น `polar`
5. เลือก `WEARABLE MODE`
6. กดเริ่มทำงานหรือสแกน เพื่อให้ระบบเรียกหน้าต่างเชื่อมต่อ Bluetooth
7. เลือกอุปกรณ์ที่ขึ้นต้นชื่อว่า Polar
8. อนุญาตการเชื่อมต่อ
9. เมื่อเชื่อมต่อสำเร็จ ระบบจะเริ่มอ่านค่า HR และคำนวณ HRV จาก RR interval

### การเชื่อมต่อ Empatica E4

1. เปิดอุปกรณ์ Empatica E4
2. เปิดโปรแกรม `E4 Streaming Server`
3. ตรวจว่า server พร้อมใช้งานตาม host/port ที่ระบบกำหนด
4. เปลี่ยนการตั้งค่าอุปกรณ์ในระบบจาก `mock` ไปเป็น `empatica`
5. เลือก `WEARABLE MODE`
6. เริ่มการสแกน
7. ระบบจะเชื่อมต่อผ่าน WebSocket ไปยัง E4 Streaming Server
8. เมื่อเชื่อมต่อสำเร็จ ระบบจะ subscribe ค่า GSR, BVP และการหายใจ
9. ระบบจะนำค่าที่ได้รับไปประเมิน HR, HRV, GSR และ RR

### การเชื่อมต่อ Muse 2 / Muse S

1. เปิดอุปกรณ์ Muse
2. เตรียม Muse SDK หรือ `muse-js` ให้พร้อม
3. เปลี่ยนการตั้งค่าอุปกรณ์ในระบบจาก `mock` ไปเป็น `muse`
4. เลือก `WEARABLE MODE`
5. เริ่มสแกน
6. ระบบจะเชื่อมต่อกับ Muse Client
7. เมื่อเชื่อมต่อสำเร็จ ระบบจะเริ่มอ่าน EEG readings
8. ระบบจะนำค่าไปคำนวณ Theta, Alpha, Beta, Gamma และตัวชี้วัดที่เกี่ยวข้อง

### การเชื่อมต่ออุปกรณ์ประเภทอื่น

1. เปลี่ยนการตั้งค่าเป็น `generic`
2. พัฒนา connector สำหรับอุปกรณ์นั้นเพิ่มเติม
3. map ค่าที่ได้เข้า `bio` และ `bands`
4. ทดสอบจนมั่นใจว่าค่าที่อ่านได้ถูกส่งเข้าระบบครบถ้วน

## 10. ระบบส่งไปทำงานยังไง
คำว่า “ส่งไปทำงาน” ในระบบนี้หมายถึงการส่งผลประเมินไปยังขั้นตอนถัดไปของการดูแล หรือส่งเข้าสู่ระบบกลาง

ระบบมีวิธีส่งออก 3 รูปแบบหลัก

### 10.1 ส่งเป็นรายงานพิมพ์
ใน Clinic Profile ผู้ใช้สามารถกด `PRINT` เพื่อสร้างรายงาน `Clinic Screening Report` สำหรับพิมพ์ออกกระดาษได้

รายงานที่พิมพ์จะมีข้อมูลหลัก เช่น

1. Subject ID
2. Profile
3. Baseline Source
4. เวลาที่ประเมิน
5. สถานะที่ประเมินได้
6. ระดับความมั่นใจ
7. ค่า HR, HRV, GSR, RR
8. ค่า Theta/Alpha
9. Summary
10. Critical Alerts
11. Findings

### 10.2 ส่งเข้าระบบกลางโดยตรง
ถ้ามีการตั้งค่า endpoint ผ่านปุ่ม `SET TARGET` ระบบจะส่งผลด้วยวิธี `HTTP POST` ไปยัง URL ที่กำหนด โดยใช้ข้อมูลแบบ JSON

โครงสร้างข้อมูลที่ส่งประกอบด้วย

1. ประเภท report
2. เวลาสร้างข้อมูล
3. Subject ID
4. บทบาทผู้รับถัดไป เช่น `NURSE_OR_DOCTOR`
5. profile mode
6. baseline source
7. profile id และ version
8. input mode
9. เวลาที่ประเมิน
10. state ที่ตรวจพบ
11. summary
12. ค่า signal หลัก
13. critical alerts
14. findings

### 10.3 ส่งออกเป็นไฟล์ JSON Handoff

2. กำลังวิเคราะห์ EEG Bands
3. กำลังตรวจ Critical Alerts

พร้อมแถบความคืบหน้าและข้อความอธิบายใต้หัวข้อ

หลังสแกนเสร็จ ระบบจะแสดงหน้าต่างผลลัพธ์สรุป และในโหมดคลินิกสามารถกด Print หรือ Send To ได้จาก popup นี้ด้วย

## 13. ระบบสามารถแสดงผลภาวะอะไรได้บ้าง
จากโครงสร้างระบบ พบว่ามีการประเมินสถานะได้หลายแบบ เช่น

1. Flow
2. Alert / Ready
3. Acute Stress
4. Confusion
5. Low Arousal
6. High Arousal
7. Fatigue / Drowsiness
8. Neutral
9. Frustration
10. Anxiety

นอกจากนี้ยังมีการตรวจ pattern หรือความเสี่ยงทางสรีรวิทยา เช่น

1. Apnea Risk
2. Hyperventilation
3. Anxiety Pattern
4. Burnout Risk
5. Chronic Stress
6. High Cognitive Load
7. Semantic Incongruity

## 13.1 ตารางสรุปภาวะที่ระบบประเมินได้พร้อมเกณฑ์โดยสังเขป

| ภาวะ | ความหมายโดยสรุป | ลักษณะสัญญาณโดยสังเขป |
|---|---|---|
| Flow | สมาธิดีและสมดุล | HR ปกติ, GSR อยู่ในเกณฑ์, HRV ดี |
| Alert / Ready | พร้อมใช้งานตามปกติ | สัญญาณชีวภาพอยู่ในช่วงปกติ, HRV ดี |
| Acute Stress | เครียดเฉียบพลัน | HR สูง, GSR สูง, HRV ลด, RR เพิ่ม |
| Confusion | สับสนหรือประมวลผลผิดปกติ | HR/GSR สูงปานกลาง, HRV ลด, Theta สูง |
| Low Arousal | ตื่นตัวต่ำ | HR ต่ำ, HRV ต่ำถึงปานกลาง, GSR ต่ำ |
| High Arousal | ตื่นตัวสูง | HR และ GSR สูงขึ้น, HRV ลดลงเล็กน้อย |
| Fatigue / Drowsiness | อ่อนล้าหรือง่วง | Theta สูง, HR/GSR ต่ำลง, ความพร้อมลดลง |
| Neutral | อยู่ในเกณฑ์ปกติ | ค่าต่าง ๆ ใกล้ baseline หรือช่วงปกติ |
| Anxiety Pattern | แนวโน้มวิตกกังวลสะสม | HRV ต่ำต่อเนื่อง, มีภาวะ ANXIETY ซ้ำ |
| Burnout Risk | เสี่ยงหมดไฟ | HRV ต่ำเรื้อรัง ร่วมกับภาวะลบสะสม |
| Chronic Stress | เครียดเรื้อรัง | STRESS ซ้ำหลายครั้ง และ HRV ต่ำต่อเนื่อง |
| High Cognitive Load | ภาระสมองสูง | Theta/Alpha ratio สูง |
| Apnea Risk | เสี่ยงหยุดหายใจ | RR ต่ำมากกว่าปกติอย่างชัดเจน |
| Hyperventilation | หายใจเร็วผิดปกติ | RR สูงเกินเกณฑ์ |

หมายเหตุ:
ตารางนี้เป็นคำอธิบายแบบย่อเพื่อใช้งานในคู่มือ ส่วนการตัดสินจริงให้ยึดผลการประเมินจากระบบและเกณฑ์ threshold ที่ตั้งไว้ในโปรไฟล์ใช้งาน

## 14. ตัวอย่างสัญญาณเตือนสำคัญ
ระบบมีการตรวจ Critical Alert โดยตรง เช่น

1. HRV ต่ำกว่าค่าเกณฑ์วิกฤติ
2. RR ต่ำมากจนเสี่ยงภาวะหยุดหายใจ
3. HR สูงมากผิดปกติ
4. GSR สูงผิดปกติจนเป็นภาวะตื่นตัวรุนแรง

เมื่อพบเหตุเหล่านี้ ระบบจะแจ้งเตือนในผลประเมินและแสดงคำแนะนำให้หยุดกิจกรรมหรือส่งต่อการดูแล

## 14.1 ตัวอย่างกรณีการส่งต่อจริง

### กรณีที่ 1 พบ HRV ต่ำกว่าค่าเกณฑ์วิกฤติ
ถ้าระบบพบว่า HRV ต่ำมากจนเข้าเกณฑ์วิกฤติ ระบบจะแจ้ง Critical Alert และแนะนำให้หยุดกิจกรรมทันที ไม่ควรฝืนทำงานต่อ จากนั้นควรให้ผู้ดูแลประเมินซ้ำ ตรวจชีพจร การหายใจ และสภาพทั่วไปของผู้ใช้งาน แล้วจึงพิจารณาส่งต่อพยาบาลหรือแพทย์

### กรณีที่ 2 พบ RR ต่ำมากจนเสี่ยงภาวะหยุดหายใจ
ถ้าระบบแจ้งว่า RR ต่ำมากผิดปกติ ให้ถือเป็นกรณีเร่งด่วน ควรหยุดการใช้งานทันที ตรวจการตอบสนองของผู้ใช้งาน จัดทางเดินหายใจ และรีบส่งต่อทีมฉุกเฉินหรือบุคลากรทางการแพทย์

### กรณีที่ 3 พบ Hyperventilation
ถ้าระบบพบการหายใจเร็วเกินเกณฑ์ ให้หยุดกิจกรรม พาผู้ใช้งานนั่งพักในที่สงบ ลดสิ่งกระตุ้น และติดตามอาการ หากอาการไม่ดีขึ้นหรือมีอาการเวียนศีรษะ หน้ามืด ใจสั่นมาก ควรส่งต่อประเมินโดยผู้เชี่ยวชาญ

### กรณีที่ 4 พบ Burnout Risk หรือ Chronic Stress ซ้ำ
ถ้าระบบพบรูปแบบความเครียดเรื้อรังหรือเสี่ยงหมดไฟต่อเนื่อง แม้จะไม่ใช่ภาวะฉุกเฉิน ควรส่งต่อเพื่อประเมินเชิงลึก เช่น ปรับภาระงาน ประเมินสุขภาพจิต หรือวางแผนฟื้นฟูร่วมกับบุคลากรที่เกี่ยวข้อง

## 15. การใช้งานในงานคลินิก
ถ้าใช้ในงานคลินิก ควรทำตามขั้นตอนนี้

1. เลือก Clinic Profile
2. ใส่ Subject ID หรือ ANON ตามนโยบายหน่วยงาน
3. ตรวจว่า baseline source เป็น `clinic_standard`
4. เชื่อมต่ออุปกรณ์จริงให้ครบ
5. กด Scan
6. ตรวจผล Assessment
7. ถ้าต้องส่งต่อ ให้กด `SEND TO`
8. ถ้าต้องแนบเอกสาร ให้กด `PRINT`

## 15.1 ขั้นตอนการส่งต่อในงานจริงแบบแนะนำ

1. สแกนและตรวจผลในแท็บ Assessment
2. ตรวจว่ามี Critical Alert หรือไม่
3. ถ้ามีภาวะผิดปกติรุนแรง ให้หยุดกิจกรรมของผู้ใช้งานทันที
4. ประเมินอาการร่วมที่สังเกตได้จริง เช่น การหายใจ สีหน้า การตอบสนอง
5. ถ้าอยู่ใน Clinic Profile ให้กด `PRINT` เพื่อพิมพ์รายงาน หรือกด `SEND TO` เพื่อส่งเข้าระบบกลาง
6. ถ้าระบบกลางยังไม่ตั้งค่า ระบบจะดาวน์โหลดไฟล์ JSON handoff ให้ใช้ส่งต่อด้วยตนเอง
7. แนบผลรายงานหรือไฟล์ handoff ไปพร้อมกับการส่งต่อผู้ใช้งานไปยังพยาบาล แพทย์ หรือผู้ดูแลลำดับถัดไป

## 16. ข้อควรระวัง

1. ถ้าใช้โหมด Wearable แต่ยังไม่เชื่อมต่ออุปกรณ์ ระบบจะอ่านข้อมูลไม่ได้
2. ถ้าใช้ Clinic Profile แต่ baseline ไม่ใช่ `clinic_standard` ระบบจะล็อกการทำงาน
3. ถ้าไม่ได้ตั้งค่า endpoint ระบบจะไม่ส่งตรง แต่จะดาวน์โหลดไฟล์ JSON แทน
4. ถ้า Popup ถูกบล็อก อาจพิมพ์รายงานไม่ได้
5. ปัจจุบันระบบตั้งค่าอุปกรณ์เป็น `mock` จึงต้องปรับก่อนใช้งานจริง

## 17. สรุปสำหรับใส่ในเอกสาร Word
ระบบ Nuengdeaw Wellness Monitor เป็นระบบประเมินภาวะผู้ใช้งานจากสัญญาณชีวภาพและ EEG โดยสามารถรับข้อมูลจากอุปกรณ์สวมใส่หลายชนิด เช่น Polar, Empatica และ Muse หรือใช้โหมดจำลองสำหรับทดสอบ ระบบแสดงผลผ่านหน้าจอแบบเรียลไทม์ มีทั้งส่วน Assessment, Dashboard, ค่าชีวภาพสด, ประวัติการสแกน และคำเตือนสำคัญ อีกทั้งยังสามารถพิมพ์รายงานในงานคลินิก ส่งผลเข้าระบบกลางผ่าน JSON หรือส่งออกข้อมูลเป็น CSV ได้ เหมาะสำหรับงาน Wellness Monitoring, การคัดกรองเบื้องต้น, และการส่งต่อข้อมูลไปยังผู้ดูแลในขั้นตอนถัดไป
